# ClassTrace 全量功能验收矩阵

本文件是重构项目的完成定义。原小程序只读；所有条目均为必做，不划分 MVP、P0、P1 或后续版本。只有每一行同时具备数据模型、API、权限、iOS 界面、异常状态和自动化验证，才可标记完成。

## 身份与用户

- 手机验证码注册、登录、退出、会话恢复与令牌刷新
- Sign in with Apple；微信历史身份迁移绑定
- 教师、家长和管理员多角色切换
- 查看和修改头像、昵称、联系方式
- 查询关联账号、重置历史账号凭据
- VIP 状态与权益查询；账号注销与数据导出

## 学生与监护关系

- 创建、查看、修改和删除学生档案
- 邀请或绑定孩子、解除绑定、切换当前孩子
- 多监护人关系、主要监护人和关系说明
- 学生详情聚合：班级、课时、考勤、作业、计划和错题

## 课程、班级与成员

- 课程模板创建、列表、详情、修改和删除
- 班级创建、列表、详情、修改、暂停、结课和删除
- 一对一、小班；预付课时和现金记账
- 邀请码信息、加入班级、申请审批、拒绝和移除成员
- 添加学生、修改单价、成员详情、班级和学生统计
- 教师课程、家长/学生已加入课程与班级

## 排课、上课与考勤

- 周期规则生成课节、日期范围查询和日历展示
- 手工课节、补课、调课、批量重新生成和冲突检测
- 课节详情、上课确认、取消、撤销确认和课后反馈
- 单人和批量考勤；出勤、请假、缺席与课时不足
- 学生/班级考勤记录、统计、修改和删除
- 确认课节、考勤、扣课时、订单履约和通知必须事务一致

## 课时账本

- 充值、消费、人工调整、退款和撤销流水
- 学生班级余额、班级历史、学生总账和可导出总账
- 低课时预警；余额不可由客户端直接覆盖
- 每笔写入支持幂等键，并保留操作者、原因和关联课节

## 教学内容

- 作业创建、发布、列表、详情、修改和删除
- 学生提交、教师批改、退回、评分和附件访问
- 教学资料保存、列表、删除、预签名上传和下载
- 文本与文件安全检查；不安全内容阻断并留审计记录

## 学习管理

- 学习首页聚合
- 学习计划创建、修改、删除、完成和每日打卡
- 打卡历史、连续天数与完成统计
- 错题创建、列表、详情、修改、删除和掌握标记
- 积分增加、积分流水、统计和成长数据

## 首页、经营与统计

- 教师和家长角色化首页
- 今日课节、待办、课时预警、未读通知和公告
- 教师课时、班级、学生、订单和收入经营概览
- 日期范围筛选；金额均使用最小货币单位存储

## 通知、公告与反馈

- 站内通知列表、未读数、单条已读和全部已读
- APNs 设备注册、偏好设置、失败重试和去重
- 课前、课后、考勤、资料与作业事件通知
- 微信历史订阅状态作为迁移兼容信息，不作为 iOS 主通道
- 公告列表、详情、创建、发布时间和受众范围
- 用户反馈提交、我的反馈、状态和客服回复

## 账单、支付与退款

- 教学订单、线下/外部支付记账、支付列表与统计
- 教师全额收款为默认结算策略；平台记录履约，不形成资金池
- 退款申请、协商记录、外部渠道退款结果和课时回退
- 支付与退款回调验签、去重和不可变事件存档
- 若启用平台支付，只能通过具备相应资质的支付机构子商户方案接入

## VIP 与系统能力

- StoreKit 2 商品、购买、恢复购买和订阅状态
- App Store Server Notifications 幂等处理
- 服务端权益授予、到期、撤销和内部赠送
- 关于、隐私政策、用户协议、版本信息和问题反馈
- 数据迁移、备份恢复、审计、可观测性和管理员工具

## 原云函数覆盖基线

必须覆盖以下 24 个原服务域：`announcement`、`attendance`、`business`、`class`、`classHour`、`contentSecurity`、`course`、`courseMember`、`feedback`、`getHomeData`、`homework`、`manualCourse`、`material`、`migrateData`、`mistakebook`、`notification`、`payment`、`plan`、`schedule`、`student`、`study`、`teacher`、`user`、`userGrowth`。

## 每项完成判定

1. PostgreSQL 模型和迁移存在，约束与索引明确。
2. REST 路由使用统一成功/错误结构，并生成 OpenAPI。
3. 教师、家长、管理员权限有正反测试。
4. 幂等、并发和状态迁移有自动化测试。
5. SwiftUI 有正常、加载、空、错误和无权限状态。
6. 从原 CloudBase 集合到新表的迁移映射明确。
7. 在 macOS 上通过 Xcode 单元测试和 UI 冒烟测试。
