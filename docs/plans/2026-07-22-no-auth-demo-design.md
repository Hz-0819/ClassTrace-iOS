# iOS 免登录功能测试分支

## 目标

在不依赖手机号、Sign in with Apple、腾讯云旧账号或公网 API 的情况下，让 TestFlight/侧载构建直接进入主界面，验证信息架构、页面跳转、表单和主要业务操作。

## 方案

分支 `codex/ios-no-auth-functional-test` 在 XcodeGen 配置中启用 `DEMO_MODE`。应用启动时注入同时具有教师、家长和管理员角色的演示用户；`HTTPClient` 保留真实网络实现，但在演示模式下将请求交给进程内 `DemoAPI`，返回覆盖首页、学生、课程、班级、课节、作业、资料、计划、错题、通知、账单、VIP、经营统计和管理功能的样例响应。文件上传在本地模拟成功，不发送真实资料。

## 安全边界

- 仅该独立分支启用 `DEMO_MODE`，不合并到正式发布分支。
- 顶部始终展示橙色演示标识。
- 演示操作不写入腾讯云、新 API、Keychain 或真实账户。
- 演示数据在应用重启后恢复初始状态。
- 正式登录、APNs 和 StoreKit 验收仍必须回到真实配置分支完成。

## 验收

1. 启动后不出现登录页。
2. 五个主导航和所有二级入口可访问。
3. 核心列表展示非空样例数据。
4. 创建、编辑、确认、打卡、已读等操作能获得成功响应或明确的演示模式提示。
5. Release 构建顶部仍显示演示标识，防止误认成生产版本。
