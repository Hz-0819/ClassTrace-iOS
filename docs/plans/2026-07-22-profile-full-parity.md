# “我的”功能完整对齐设计

## 目标

“我的”不是菜单集合，而是教师和家长的业务总入口。所有展示卡片必须来自本地仓库数据，所有入口必须进入可操作页面，修改结果必须经过统一 Repository 写入并在演示模式持久化。

## 教师端

- 顶部展示教师身份、账号短 ID、班级数、学生数与累计/剩余课时，并支持在已开通角色之间切换。
- “我的班级”预览直达班级详情；班级管理、学生管理、作业管理、资料中心和个人日程均进入完整页面。
- 经营概览支持月份选择、收入确认进度、课节执行、班级收入构成与低课时提醒。
- 课时档案支持班级汇总、学生级课时流水和 CSV 分享导出。
- 账单中心仅允许教师确认外部收款、处理退款，不在应用内代收课费。

## 家长端

- 顶部展示家长身份、孩子数、在读课程与积分，并支持角色切换。
- 孩子预览直达完整档案；档案包含课程、课时余额、考勤、作业、学习计划、错题及监护关系。
- 孩子管理支持新增、邀请码绑定、编辑、删除和逐班级课时明细。
- 账单中心允许建立双方线下课费记录和申请退款，不允许执行教师侧收款确认。

## 共用与扩展功能

- 积分页对齐总积分、今日/本周/本月统计、连续活跃天数和全部/打卡/作业/考勤筛选。
- 反馈页区分图片和文件附件，支持预览/删除、500 字限制、提交成功、再提一条、返回及历史回复。
- 通知支持逐条/全部已读和分类开关；账号设置支持资料、角色、手机、通知、数据导出与注销。
- VIP 保持 App Store 数字权益与激活码逻辑；关于页提供功能说明、联系方式复制、隐私政策和用户协议。

## 验证标准

1. 不存在空操作按钮或只显示提示的主要入口。
2. 教师与家长看到的账单操作权限不同。
3. 新增、编辑、删除、绑定、确认、退款、通知设置、反馈和积分数据在本地演示数据库中可复现。
4. 只在全部代码完成后触发一次 GitHub iOS 构建、单元测试和架构检查。
